## 概述

**扣扣快打**支持多种方式获取抖音直播弹幕：

- **方式一：真实WebSocket连接** (推荐)
  - 优点：实时、准确、完整
  - 缺点：需要Cookie，可能被风控

- **方式二：演示模式** (测试用)
  - 优点：无需配置，即开即用
  - 缺点：数据是模拟的

- **方式三：浏览器扩展** (可选)
  - 优点：无需Cookie，稳定
  - 缺点：需要安装扩展

## 获取Cookie

### 步骤详解：

1. **打开抖音网页版**
   - 访问: https://live.douyin.com
   - 使用手机抖音扫码登录

2. **打开开发者工具**
   - Windows/Linux: 按 `F12` 或 `Ctrl + Shift + I`
   - Mac: 按 `Cmd + Option + I`

3. **切换到 Network 标签**
   - 点击顶部的 "Network" 标签
   - 刷新页面 (F5 或 Cmd+R)

4. **找到任意请求**
   - 在 Network 列表中，点击任意请求
   - 查看右侧的 "Headers" 部分

5. **复制 Cookie**
   - 在 "Request Headers" 中找到 "Cookie" 字段
   - 点击 Cookie 旁边的值，全选并复制

6. **保存Cookie**
   - 将复制的Cookie粘贴到"抖音配置"页面的Cookie输入框中
   - 点击"保存Cookie"按钮

### Cookie 示例：

```
sessionid=abc123; ttwid=def456; passport_csrf_token=ghi789; ...
```

**重要提示：**
- Cookie 包含您的登录信息，请妥善保管
- Cookie 可能会过期，需要定期更新
- 不要将Cookie分享给他人

## 获取直播间ID

### 方法一：从URL获取

1. 打开任意抖音直播间
2. 查看浏览器地址栏
3. URL格式：`https://live.douyin.com/123456789`
4. 其中 `123456789` 就是直播间ID

### 方法二：从分享链接获取

1. 点击直播间的"分享"按钮
2. 选择"复制链接"
3. 从链接中提取房间ID

### 方法三：从页面源码获取 (高级)

1. 在直播间页面按 F12
2. 切换到 Console 标签
3. 输入以下命令：
   ```javascript
   location.pathname.split('/').pop()
   ```
4. 输出的就是房间ID

## 配置应用

### 步骤：

1. **打开抖音配置页面**
   - 点击侧边栏的"抖音配置"

2. **配置Cookie**
   - 粘贴您获取的Cookie
   - 点击"保存Cookie"
   - 点击"测试Cookie"验证格式

3. **配置直播间ID**
   - 输入直播间ID
   - 点击"保存房间ID"

4. **测试连接**
   - 点击"测试连接"
   - 查看连接状态
   - 查看测试弹幕列表

5. **开始使用**
   - 返回"扣扣快打"主页面
   - 选择"真实模式"
   - 点击"开启自动打单"

## 常见问题

### Q1: 连接失败怎么办？

**A:**
1. 检查 Cookie 是否有效
2. 检查网络连接
3. 尝试重新获取 Cookie
4. 确认直播间是否在直播中

### Q2: Cookie 多久过期？

**A:**
- 通常有效期约 7-30 天
- 频繁登录可能缩短有效期
- 建议每周检查一次

### Q3: 被风控了怎么办？

**A:**
1. 立即停止使用
2. 等待一段时间 (通常1-24小时)
3. 重新获取 Cookie
4. 降低使用频率

### Q4: 弹幕有延迟怎么办？

**A:**
- WebSocket 通常有 1-3 秒延迟
- 这是正常现象
- 确保网络稳定

### Q5: 可以同时监听多个直播间吗？

**A:**
- 当前版本支持单个直播间
- 可以切换直播间
- 多直播间功能在开发中

## 技术细节

### WebSocket 连接流程

```
1. 生成签名 (X-Bogus)
2. 建立 WebSocket 连接
3. 发送认证信息
4. 接收弹幕数据
5. 解析 Protocol Buffer
6. 提取用户和内容
7. 匹配扣数规则
8. 触发打印
```

### 依赖的npm包

- `protobufjs` - Protocol Buffer 解析
- `pako` - 数据解压 (如需要)
- `axios` - HTTP 请求
